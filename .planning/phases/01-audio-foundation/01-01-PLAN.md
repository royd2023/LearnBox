  <name>Task 2: Implement learnbox/mic.py with energy-based VAD</name>
  <files>learnbox/mic.py</files>
  <action>
    Create learnbox/mic.py implementing microphone capture with energy-based VAD.

    Module-level constants:
    ```python
    SAMPLE_RATE = 16000       # Hz — required for Moonshine compatibility (Phase 2)
    CHANNELS = 1              # mono
    DTYPE = "int16"           # Moonshine needs float32 but STT layer handles conversion
    CHUNK_FRAMES = 1600       # 100ms per chunk at 16kHz
    DEFAULT_SILENCE_RMS = 300 # int16-scale RMS; tune per environment
    SILENCE_CHUNKS = 10       # 1.0s consecutive silence = end of speech
    MAX_RECORD_CHUNKS = 150   # 15s max recording cap — prevents unbounded capture
    ```

    Implement these functions:

    **`list_devices() -> None`**
    Print sd.query_devices() output. Used for remote Pi debugging. No parameters.

    **`calibrate_silence(duration_s: float = 1.0) -> int`**
    Sample `duration_s` seconds of silence from the default input device.
    Return int(2 * ambient_rms) rounded up, clamped to a minimum of 100.
    Opens an InputStream for the specified duration, reads all chunks, computes overall RMS.
    Prints the measured ambient RMS and the resulting threshold for user visibility.
    Use this before record_until_silence() in noisy environments.

    **`record_until_silence(silence_threshold: int = DEFAULT_SILENCE_RMS, silence_duration_chunks: int = SILENCE_CHUNKS) -> np.ndarray`**
    Core VAD capture loop. Returns int16 mono numpy array at SAMPLE_RATE.

    Implementation rules:
    - Never pass device= to InputStream. Use the system default.
    - samplerate=SAMPLE_RATE, channels=CHANNELS, dtype=DTYPE always — do not use device defaults.
    - Read CHUNK_FRAMES at a time in a while loop.
    - Compute RMS: `float(np.sqrt(np.mean(chunk.astype(np.float32) ** 2)))`
    - State machine:
      - Before speech: if rms >= silence_threshold → set speech_started=True, reset silent_count=0, append chunk
      - During speech: always append chunk; if rms < silence_threshold → increment silent_count; if silent_count >= silence_duration_chunks → break
      - Pre-speech silence frames are NOT appended (avoid feeding leading silence to Moonshine)
    - If len(frames) reaches MAX_RECORD_CHUNKS → break (15s safety cap)
    - If no frames accumulated (speech never started) → return np.zeros(0, dtype=np.int16)
    - Return: np.concatenate(frames, axis=0).flatten()
    - If InputStream raises PortAudioError (no device available) → let it propagate; do not swallow.

    IMPORTANT dtype note: mic.py intentionally returns int16. Phase 2 stt.py will convert:
    `audio_float32 = audio_int16.astype(np.float32) / 32768.0`
    Add this as a docstring note in record_until_silence() so the Phase 2 implementer sees it.

    Add module docstring explaining: "Audio capture with energy-based VAD. Returns int16 at 16kHz mono. Caller (stt.py) is responsible for float32 conversion before Moonshine."
  </action>
