  <name>Task 1: Implement learnbox/audio.py with blocking playback</name>
  <files>learnbox/audio.py</files>
  <action>
    Create learnbox/audio.py implementing speaker playback.

    Module docstring: "Speaker playback via sounddevice. Platform-neutral — same code runs on Windows and Pi 5. Never hardcode device index."

    Implement one function:

    **`play_audio(audio: np.ndarray, sample_rate: int) -> None`**

    Implementation rules:
    - `sd.play(audio, samplerate=sample_rate)` followed immediately by `sd.wait()`
    - sd.wait() is non-negotiable — without it, the function returns before audio finishes, which causes pipeline overlap bugs in Phase 3.
    - Never pass device= to sd.play(). Use the system default output.
    - Accept both int16 and float32 numpy arrays — sounddevice handles both. Do not force a conversion.
    - Docstring must note: "Blocks until playback is complete. audio can be int16 or float32. sample_rate must match the audio content (e.g., 16000 for captured mic audio, 22050 for Piper TTS output in Phase 3)."

    Do not add a last_audio buffer or replay feature here — that belongs in Phase 3 pipeline.py.

    Anti-patterns to avoid:
    - sd.play(audio) without sd.wait() — non-blocking, causes pipeline overlap
    - sd.play(audio, device=0) — hardcoded device, breaks on any machine where device 0 is not output
    - threading.Thread for playback — unnecessary complexity; sd.wait() is already blocking
  </action>
